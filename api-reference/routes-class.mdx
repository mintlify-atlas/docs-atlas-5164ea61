---
title: Routes class
description: URL builder for all rybbit API endpoints, exported for direct use or advanced integrations.
---

The `Routes` class constructs typed `URL` objects for every rybbit API endpoint. It is used internally by the [`Rest`](/api-reference/rest-class) class, but is also exported from `rybbit.ts` for cases where you want to build URLs without triggering an HTTP request — for example, when you need to pass a URL to your own fetch wrapper.

## Import

```typescript
import { Routes } from 'rybbit.ts';
```

## Constructor

```typescript
constructor(base: URL)
```

<ParamField path="base" type="URL" required>
  The base URL of your rybbit instance. All generated URLs are derived from this origin.
</ParamField>

```typescript
const routes = new Routes(new URL('https://app.rybbit.io'));

// Build a URL without making a request
const url = routes.getOverview(42, { startDate: '2024-01-01' });
console.log(url.toString());
// https://app.rybbit.io/api/sites/42/overview?start_date=2024-01-01
```

<Tip>
  When using the `Rest` class, a `Routes` instance is available at `client.routes`. You rarely need to construct one yourself.
</Tip>

## Methods

All methods return a `URL`. Query parameters with `undefined` values are omitted automatically. `filters` arrays are serialized as JSON strings.
